# SMASプロジェクトに必要な理論分野 一覧

## 中核(①〜⑤層に直接対応、習得優先度: 高)

| 分野 | 対応する層 | 具体的に何を使うか |
|---|---|---|
| **ゲーム理論(非協力ゲーム)** | ②誘因構造層 | 支配戦略、ナッシュ均衡、耐戦略性の基礎定義 |
| **メカニズムデザイン** | ②誘因構造層 | VCGメカニズム、誘因整合性、収益同値定理 |
| **計算論的社会選択理論** | ③集約層 | 投票理論、マッチング理論、公正分配、操作耐性の証明 |
| **応用圏論(Applied Category Theory)** | ⑤検証層 | モノイダル圏、ストリング図、関手、合成則(結合律・単位律) |
| **情報理論** | ①環境層/①概念層 | 情報の非対称性、必要多様性の法則(Ashby)の定量化 |

## 実証・評価手法(習得優先度: 高〜中)

| 分野 | 用途 |
|---|---|
| **マルコフ連鎖・マルコフ決定過程** | 収束確率・収束速度の評価 |
| **形式手法(TLA+/Quint)** | 並行安全性・競合状態の検証 |
| **確率論・統計的検定の基礎** | モンテカルロ試行の設計、収束率の有意性判断 |

## 実行主体層の周辺(習得優先度: 中、拡張フェーズ)

| 分野 | 用途 |
|---|---|
| **部分観測マルコフ決定過程(POMDP)** | 情報の非対称性を持つエージェントの意思決定モデル |
| **強化学習の基礎** | マルチエージェント環境(PettingZoo等)でのエージェント実装 |
| **繰り返しゲーム理論** | 信用枠配分・評判形成という時間発展ケースの設計 |

## 隣接・背景知識(習得優先度: 中〜低、"なぜこの設計か"を語るための土台)

| 分野 | 用途 |
|---|---|
| **サイバネティクス(特にAshbyの必要多様性の法則)** | ①概念層の理論的背景、プロジェクト全体のメタ理論としての位置づけ |
| **コストリー・シグナリング理論(ゲーム理論の一分野)** | 痕跡の偽装コスト設計 |

## 将来の拡張候補(現時点では優先度低・保留)

| 分野 | 保留理由 |
|---|---|
| **代数的位相幾何学(組合せ論的トポロジー)** | 合意形成の不可能性証明。習得コストが高く、今回はスコープ外(⑤検証層の"より厳密な将来版") |
| **計量セマンティクス・確率的ドメイン理論(Denotational Semantics / Probabilistic Domain Theory)** | マルコフ決定過程による収束確率の"実証的評価"に対し、不動点定理により"必ず収束する"ことを理論的に証明する、より厳密な版。アーキテクトとしての説得力にはMDPの実証評価で十分なため、研究レベルの厳密性が必要になった時のみ着手 |
| **パーコレーション理論・グラフ動態** | 痕跡密度の臨界点分析。1ケース目の規模では不要 |
| **自由エネルギー原理/アクティブインファレンス** | ④実行主体層の一実装オプション(POMDPの一般化)、拡張フェーズ |
| **オートポイエーシス理論** | 新しい層ではなく、参加者の入退場・委任連鎖という評価観点を考える際の解釈のレンズ |

## 意図的にスコープ外とした分野(参考・境界線の明示用)

| 分野 | SMASでは扱わない理由 |
|---|---|
| **協力ゲーム理論(シャープレイ値・コア等)** | 「提携形成」という別トピック。今回は非協力ゲームに一貫する方針 |
| **ビザンチン故障耐性(BFT)理論** | 「悪意ある壊れた主体」への対応で、誘因設計とは異なる問題設定 |
| **アクターネットワーク理論(ANT)** | 数学的検証ができない解釈フレームワーク |

## 学習の優先順位(実務的な推奨)

1. **メカニズムデザイン+ゲーム理論の基礎**(②誘因構造層の核。ここが一番"重い"というRIUHさんの自己評価と一致するため最優先)
2. **応用圏論の実務的な使い方(DisCoPyを動かしながら)**(⑤検証層、理論より先に手を動かして体感するのが効率的)
3. **計算論的社会選択理論(`pref_voting`のドキュメントを読みながら)**(③集約層)
4. **マルコフ決定過程・収束性評価の基礎**(評価キットの実装に直結)
5. 以降、拡張フェーズで POMDP・繰り返しゲーム理論・サイバネティクスの背景理解へ
