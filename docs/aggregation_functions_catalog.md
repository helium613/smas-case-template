# MASにおける集約関数の一覧

複数主体の入力(申告・選好・状態)を、1つの結論・状態に変換する関数を「集約関数」として広く捉え、出自の異なる分野ごとに整理する。

---

## 1. メカニズムデザイン由来(support: SMASの③集約層で既に採用)

| 集約関数 | 何を集約するか |
|---|---|
| VCGの配分ルール | 各主体の評価額の申告 → 誰が資源を得るか |
| 第二価格ルール | 入札額の申告 → 支払い額 |
| Gale-Shapleyのマッチング | 双方向の選好リスト → 安定マッチング |
| Top Trading Cycle | 初期保有と選好 → 効率的な交換結果 |

## 2. 計算論的社会選択理論由来(既にpref_votingで採用)

| 集約関数 | 何を集約するか |
|---|---|
| 多数決・ボルダ得点 | 各主体の順位付け → 単一の勝者・順位 |
| コンドルセ方式 | ペア比較の集合 → 全体順位 |
| 調整されたウィナー法 | 2者間の選好強度 → 妬みのない資源分配 |

## 3. 平均場理論(Mean Field Theory)由来 ※新規追加

多数のエージェントが存在する極限で、個々の相互作用を「全体の平均的な状態(場)」との相互作用に近似する集約手法。物理学(統計力学)発。

| 集約関数 | 何を集約するか |
|---|---|
| 平均場ゲーム(Mean Field Games, MFG) | 無数の類似エージェントの戦略分布 → 「代表的な1エージェントが、集団全体の分布を所与として最適反応する」という近似均衡 |
| 平均場強化学習(Mean Field MARL) | 多数のエージェントの行動分布 → 各エージェントが「近傍の平均行動」のみを観測して学習する近似 |

**特徴**: エージェント数が非常に多い(数百〜数万)場合、個々のペアワイズな相互作用を全て計算するのは非現実的。平均場近似は「集団を1つの分布として集約し、個体はその分布とだけ相互作用する」という、**計算量を劇的に下げる集約手法**。SMASのメカニズムカタログでは「参加者数が少数〜中規模」を想定していたため、平均場理論はスコープ外だったが、**将来"通信密度の臨界点"(パーコレーション理論)を検討する規模に達したら、平均場理論も同時に検討候補になる**関係にある。

## 4. 群知能・スティグマジー由来(以前の議論で触れた領域)

| 集約関数 | 何を集約するか |
|---|---|
| フェロモン濃度の重み付き和 | 個々の痕跡(局所的な行動の跡) → 経路選択の確率分布 |
| 群平均・群中心(Boids的な集約) | 近傍エージェントの位置・速度 → 自分の次の行動ベクトル |
| PageRank型再帰的評判 | 隣接ノードからの評価の連鎖 → 各ノードの評判スコア(固有値問題として解く集約) |

**実例**: C2C評判ネットワーク(Amazon・食べログ・Uberのレビュー/評価)。プラットフォームが集約アルゴリズムを設計している限りSMAS本体の1ケースとなるが、設計者不在の自然発生的な口コミは創発的秩序システムに分類される(境界事例の詳細は`architecture_family_map.md`2.5節を参照)。

## 5. 分散システム・合意形成由来

| 集約関数 | 何を集約するか |
|---|---|
| 多数決型ビザンチン合意(PBFT等) | 各ノードの提案 → 単一の合意結果(ただし前提はSMASと異なり、耐戦略性ではなく耐障害性が目的) |
| Raftのリーダー選出 | 各ノードの投票 → 単一のリーダー(SMASの「権力集中の不在」とは逆に、意図的に中央を1つ選ぶ集約) |
| CRDT(Conflict-free Replicated Data Type) | 複数ノードでの並行更新 → 順序に依存せず収束する単一の状態(可換性・結合性を保証する集約という点で、SMASの⑤検証層の関心と直接重なる) |

## 6. 統計的集約(機械学習・アンサンブル由来)

| 集約関数 | 何を集約するか |
|---|---|
| フェデレーテッドラーニングの重み平均(FedAvg) | 各クライアントで学習されたモデルパラメータ → グローバルモデル |
| アンサンブル投票(ブースティング・バギング) | 複数モデルの予測 → 単一の予測 |
| ベイズ的統合(Bayesian Model Averaging) | 複数モデルの事後確率 → 統合された予測分布 |

**特徴**: 「エージェント」というより「モデル」の集約だが、**「複数の独立した情報源を、1つの結論に変換する」という構造は同じ**。SMASの③集約層が要求する「決定論性」「局所計算・大域整合」という性質を、機械学習の文脈でどう保証するか、という応用の橋渡しが考えられる。

## 7. 物理学・複雑系由来

| 集約関数 | 何を集約するか |
|---|---|
| 秩序パラメータ(Order Parameter、相転移理論) | 個々のスピン・粒子の状態 → 系全体の「秩序度合い」を表す単一の指標 |
| くりこみ群(Renormalization Group)的な粗視化 | ミクロな詳細 → マクロなスケールでの有効な記述 |

**特徴**: これは「集約関数」というより「集約という操作そのものの理論」。ミクロからマクロへの移行(前回確認した「信頼委任システムからSMASへの創発」)を、数理的に最も厳密に扱ってきた分野の一つ。

## SMASの③集約層への位置づけ

| カテゴリ | 現在の採用状況 |
|---|---|
| 1. メカニズムデザイン由来 | **採用済み**(主軸) |
| 2. 社会選択理論由来 | **採用済み**(`pref_voting`) |
| 3. 平均場理論 | **未採用**。エージェント数が少数〜中規模の1ケース目には不要。将来、大規模化した際の拡張候補(パーコレーション理論と同格) |
| 4. 群知能・スティグマジー | **部分的に採用**(①環境層の痕跡設計の着想源として) |
| 5. 分散システム・合意形成 | **意図的に不採用**。前提(耐戦略性 vs 耐障害性)が異なるため、SMASのスコープ外(耐障害協調システムという別ファミリー) |
| 6. 統計的集約 | **未採用**。ただしLLMエージェントの判断を統合する場面(複数エージェントの提案をアンサンブル的に統合する等)で、将来接続しうる |
| 7. 物理学・複雑系由来 | **未採用**。メタ理論として、ミクロ/マクロの創発を語る際の理論的背景(2.2節の議論と関連) |

**結論**: SMASの③集約層は、現状「メカニズムデザイン+社会選択理論」という、耐戦略性・操作耐性を厳密に問える集約関数に絞って採用している。他の集約関数群(平均場、群知能、分散合意、統計的集約、物理学的集約)は、**"集約"という同じ操作を扱っていても、耐戦略性という性質を保証する設計にはなっておらず、SMASの①概念層の要求(誘因整合性・耐戦略性)を満たすには、追加の検証が必要**になる。将来の拡張時には、この一覧を「候補プール」として参照する。
