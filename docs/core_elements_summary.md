# プロジェクトの中核要素(サマリー)

## 最重要(コンセプトの心臓部)

1. **①概念層: 中核3性質+前提背景+メタ理論**
   中核3性質: 誘因整合性・耐戦略性・個人合理性(耐戦略性は誘因整合性の最強形であり包含関係にある)
   前提背景: 情報の非対称性 / メタ理論(比喩的援用): 必要多様性の法則
   → 数学的に確立された"ものさし"。これが揺らがない限りプロジェクトの根は崩れない

2. **②誘因構造層: 誘因構造設計そのもの**
   中央の執行者なしに、正直な行動が支配戦略になるよう設計する中核作業
   → RIUHさんが担う"設計者としての判断"が最も色濃く出る部分

## サブコンポーネント(①②を支える構造)

3. **5層構造(②骨格層)**
   環境・誘因構造・集約・実行主体・検証。ケースを問わず再利用できる型
   → デザインパターンとしての普遍性の器

4. **圏論による合成可能性の検証(⑤検証層)**
   合成則(結合律・単位律)のチェック、関手による新ケースへの対応づけ
   → 「構造が矛盾なく閉じているか」を機械的に確認する、他に見当たらない独自の検証手段

## 独自性を支えるもう一本の柱

5. **評価手法(検証可能性)の確立**
   Quint(TLA)による並行性・安全性検証+Python(マルコフ決定過程)による収束確率評価
   → 「言っているだけ」ではなく「動かして・測って示せる」という再現可能な証拠。理論(4)が形式的検証、こちら(5)が実行時の実証的検証という、性質の異なる2種類の裏付けを両方持っている点が強み

---

## 全体の関係図

```
①3性質+前提背景(なぜ良い設計か、のものさし)
        ↑ 満たすように設計
②誘因構造(何を設計するか、の中核)
        ↑ 実装の型として整理
③5層構造(どう構造化するか、の骨格)
        ↑ 構造の整合性を確認
④圏論的検証(構造が矛盾ないか、の証明)
        ↑ 実際に動かして確かめる
⑤評価手法/検証キット(本当に機能するか、の実証)
```

## 一言で言うなら

**「非中央集権的協調が成立するための3性質(誘因整合性・耐戦略性・個人合理性)を、5層構造として設計し、層間の型接続の整合性を機械的に検証しつつ、実行時の安全性・収束性も実証する、アーキテクチャパターンとその検証方法論」**

## この5要素のうち、独自性の核はどこか

- **1(中核3性質)**: 独自ではない(既存の数学)。ただし土台として必須
- **2(誘因構造設計)**: メカニズムデザインという既存分野の応用。独自性は中
- **3(5層構造)**: **独自性が高い**。既存のOSI的モデル(信頼できる主体前提)にはない、信頼できない主体同士を前提にした構造化
- **4(圏論的検証)**: **独自性は限定的(下方修正、`DECISIONS.md` D-08/D-14)**。要素技術(DisCoPy等)は既存であり、圏論×ゲーム理論にはCompositional Game Theory (Open Games)という確立した先行分野が存在する(`SMAS_theorymap.md` 7章)。検証力も型接続の範囲に限定される(CLAUDE.md 5章)。「他に見当たらない」とは主張しない
- **5(評価手法の確立)**: **独自性が高い**。理論(圏論)と実証(Quint/MDP)を両方揃えて「言うだけでなく測れる」形にしている点は、多くの類似プロジェクト(学術的な理論提示、または実務的な実装のみ)のどちらとも違う、両方を橋渡しする位置づけ

→ **3・4・5の組み合わせこそが、最も他に見当たらない差別化ポイント**
